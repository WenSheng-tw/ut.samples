Tutor

In this section we can see 2 ways of working.

  • Creating our web outputs from functions compiled together with our exe server
  • Creating our outputs from html files, being able to embed all our code similar to how php works.
Both ways are completely valid and functional. Although, the first one has all the code embedded in the server, this implies that no one can see it, while the second option, whoever has access to the server can see the code. It is not a handicap because systems with php work in this way and nothing happens, because in principle nobody has access to the server.

On the other hand, the second way of working avoids having to compile every time we change the code at development time, since we compile on-fly. It really is my favorite option, once you try it you can not forget it.

And for the most expert, they can route to a function that we have in the exe (controller) that when it has processed the data, execute an html (view) passing it the parameters. All the logic of the program is embedded in the exe without anyone being able to see it.

In summary we have several ways of working. Enjoy it !
