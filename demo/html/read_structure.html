App structure

 Summary

We start from the basic template for this example. When you create an application with UT you will have this basic structure:



App
Folder where the prgs of our application will be found. These files will be compiled when we create our application. Each file must be defined within the app.hbp file that is located in the root of our project.

Files In this folder there are support files for UT that are javascript libraries. We should not touch them or modify them. We can add new javascript libraries for our use

Html Folder where all the views with extension *.prg or *.html will be found. By default these files behave the same as any html file, and it will not be necessary to compile them every time changes are made.

Lib Folder where the libraries to compile our server will be found. By default there is a version in msvc64 and another in mingw64

In the main folder of the app there are 2 bats to compile according to the library we want to use and the app.hbp file in which the prgs of the app folder we want to add are defined. Remember to adjust the paths.

Once the application is compiled and finished we should only use it for distribution:



The best way is to try compiling the first basic example of the template and see what each file does.


Enjoy it!
